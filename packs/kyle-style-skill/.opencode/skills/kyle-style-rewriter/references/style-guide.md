# Kyle Style Guide

## 1. Core style

This writing style is not decorative. It is a method for turning a problem into a structured judgment.

The preferred pattern is:

```text
background → problem → decomposition → analysis → necessity or direction
```

The text should help the reader understand:

1. What problem is being discussed.
2. Why the problem exists.
3. Which dimensions matter.
4. What judgment can be made.
5. What should be done next.

## 2. Preferred Chinese expressions

Use these when they fit the logic:

- 问题的关键在于……
- 具体来说，……
- 另一方面，……
- 在这一条件下，……
- 因此，有必要……
- 这并不意味着……而是……
- 该方法可以解决……，但仍然面临……
- 综上所述，……

## 3. Preferred English expressions

- The key issue is that...
- Specifically,...
- In practice,...
- This does not mean that...
- Therefore, it is necessary to...
- Based on the findings above,...
- The current approach can address..., but it still faces...

## 4. Chinese technical writing pattern

For formal Chinese documents:

```markdown
## 背景
[只写与问题相关的背景]

## 问题分析
[拆成2-4个问题]

## 对策或建议
[提出可执行方向]

## 小结
[收敛到必要性或下一步，不做宏大总结]
```

## 5. English technical email pattern

```text
Hi [Name],

Thanks for [context].

Based on the information available so far, my understanding is that [core conclusion].

Here are the key points:

1. [Finding]
2. [Finding]
3. [Implication]

Therefore, [next step / recommendation].

Please feel free to let me know if you have any questions or concerns.

Regards,
[Name]
```

## 6. Common replacements

| Avoid | Prefer |
|---|---|
| 赋能 | 支持 / 提供能力 |
| 普惠 | 降低门槛 / 扩大适用范围 |
| 拔高 | 提升能力 / 提高效率 |
| 民主化 | 降低使用门槛 |
| 完整闭环 | 管理链路 / 反馈机制 / 闭环管理 |
| 重塑格局 | 改变实施方式 / 改变能力边界 |
| 银弹式 | 单点方案 / 单一方案 |
| 蜂群式 | 大规模并行 / 多实例并发 |

## 7. Closing principles

A conclusion should do one of three things:

1. State necessity.
2. Define next step.
3. Clarify boundary.

Avoid:

- slogan-like ending
- emotional ending
- repeating the introduction
